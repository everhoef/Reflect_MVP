Task 13: Add fallback behavior for missing/unavailable anchors

IMPLEMENTATION
==============
Changed: frontend/src/components/retro/Coachmark.tsx

Old behavior: when resolveAnchorPosition returned null (anchor absent), Coachmark
rendered at a fixed bottom-right position (bottom:24, right:24) as a floating orphan.

New behavior: when position === null, Coachmark returns null immediately — no DOM
output, no orphaned overlay, no console errors.

Key change (lines 98-99):
  if (!visible) return null;
  if (position === null) return null;

Secondary cleanup: removed optional-chaining (?.) from position.placement references
in the render path, since position is now guaranteed non-null at render time.
